## Session Script

### Opening (5 minutes)

"Thank you for participating in this usability session. We're testing LibreAssistant, an open-source AI assistant, to understand how new users experience the interface and features.

**Important reminders:**
- We're testing the software, not you
- There are no wrong answers or actions
- Please think aloud as you work
- Feel free to ask questions or express confusion
- This session is being recorded for analysis purposes only

Do you have any questions before we begin?"

### Think-Aloud Protocol (5 minutes)

"Throughout this session, please verbalize your thoughts as you interact with the interface. Tell us:
- What you're looking for
- What you expect to see or happen
- Any confusion or frustration you experience
- What you're trying to accomplish

Let's practice with this simple example: [Show practice screen/interface]"

## Task Scenarios

### Task 1: First Impressions and Getting Started (15 minutes)

**Setup**: Show LibreAssistant main interface without any introduction.

**Instructions**: "Imagine you've just discovered LibreAssistant and want to try it out. Take a look at this interface and explore what you think it can do."

**Observe and Note**:
- Initial reactions and comments
- Where they look first
- How they explore the interface
- What they try to click on
- Questions they ask

**Probing Questions** (only if participant gets stuck):
- "What do you think this application does?"
- "What would you try first?"
- "What does [specific element] mean to you?"

**Success Criteria**:
- [ ] Participant understands this is an AI assistant
- [ ] They identify how to send a message
- [ ] They understand provider selection (at basic level)
- [ ] They attempt to interact with the interface

### Task 2: Having a Conversation (20 minutes)

**Instructions**: "Your goal is to have a simple conversation with the AI assistant. Try asking it to help you with something you might actually use an AI assistant for."

**Observe and Note**:
- How they approach provider selection
- Understanding of API key requirements
- Reaction to error messages or feedback
- How they interpret AI responses
- Whether they continue the conversation

**Probing Questions** (if needed):
- "What do you think about this [error message/response]?"
- "What would you expect to happen when you click that?"
- "How does this compare to other AI assistants you've used?"

**Success Criteria**:
- [ ] Participant successfully sends at least one message
- [ ] They understand basic provider concepts
- [ ] They can interpret system feedback
- [ ] They demonstrate understanding of conversation flow

### Task 3: Exploring Plugins (20 minutes)

**Instructions**: "You've heard that LibreAssistant has 'plugins' that add extra capabilities. Explore what plugins are available and try using one that interests you."

**Observe and Note**:
- How they discover the plugin system
- Understanding of what plugins do
- Process of enabling/installing plugins
- Success using a specific plugin
- Confusion about plugin concepts

**Probing Questions**:
- "What do you think plugins do?"
- "How would you describe this plugin to a friend?"
- "What would you expect to happen if you enable this?"

**Success Criteria**:
- [ ] Participant finds the plugin area
- [ ] They understand the concept of plugins
- [ ] They successfully enable at least one plugin
- [ ] They attempt to use an enabled plugin

## Post-Task Interview (20 minutes)

### Overall Experience

1. "How would you describe your overall experience using LibreAssistant?"
2. "What was the most frustrating or confusing part?"
3. "What worked better than you expected?"
4. "On a scale of 1-10, how likely would you be to use LibreAssistant again?"

### Specific Feature Feedback

5. "How intuitive was the process of getting started?"
6. "What did you think of the plugin system?"
7. "How helpful were the error messages and feedback you received?"
8. "What did you think of the customization options?"

### Comparative Questions

9. "How does LibreAssistant compare to other AI assistants you've used?"
10. "What features were you expecting that you didn't find?"
11. "What would you change about the interface if you could?"

### Wrap-up

12. "Is there anything else you'd like to share about your experience?"
13. "Do you have any questions about LibreAssistant that we haven't covered?"
